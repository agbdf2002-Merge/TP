# Enterprise Technology Governance Framework (ETGF)

## Purpose

This directory is the controlled repository root for ETGF Version 1.0 publication, governance engineering, implementation, assurance, executive reporting, and sustained operation.

## Governance Boundary

- Consume `ETGF-BASELINE-1.0` as the architecture boundary.
- Do not redesign governance bodies or decision rights through publication activities.
- Policy governs.
- Standards implement.
- Procedures execute.
- Evidence proves operation.
- Enterprise Architecture owns the ETGF work-package structure and architecture repository model.

## Publication Structure

| Volume | Repository Path | Purpose |

| Volume III | `03-governance-engineering/` | Requirements, controls, regulatory traceability, taxonomy, object model, and relationship model |

| Volume V | `05-implementation-operations/` | Implementation handoff, operating model, backlog, procedures, and sustainment |
| Volume VI | `06-assurance/` | Assurance, control testing, regulatory readiness, resilience, and evidence |
| Volume VII | `07-executive-appendices/` | Executive reporting, registers, templates, glossary, and publication appendices |

## Repository Rules

1. Approved identifiers shall be stable and shall not be reused.
2. Markdown is the source-controlled working format.
3. Word documents are publication artifacts generated from controlled source content.
4. Binary publication files shall be released through the approved release workflow or controlled document repository.
5. Requirements, controls, evidence, exceptions, lifecycle records, and architecture decisions shall remain traceable.
6. Repository changes shall preserve the approved ETGF architecture.
